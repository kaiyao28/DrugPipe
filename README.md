DrugPipe is a modular post-GWAS target-discovery workflow and reference code
library. It helps connect standard genetic, molecular, biological, MR, safety
and druggability evidence into ranked target hypotheses.

The current package focuses on reproducible local-file interfaces: schemas,
validation, example data, evidence-table parsers, target scoring, Markdown
reports and target evidence cards.

## Workflow

![DrugPipe workflow](docs/assets/drugpipe-workflow.svg)

DrugPipe starts from GWAS summary statistics, defines associated loci, maps
loci to candidate genes, imports evidence layers generated by appropriate
specialist tools, and integrates those layers into ranked target hypotheses.

## Design Philosophy

DrugPipe is not primarily a one-command production pipeline. It is a practical
framework for organising post-GWAS target-discovery evidence.

DrugPipe does not assume that all analyses can or should be run in a single
process. Heavy analyses can be performed independently using established tools
and appropriate compute resources, then imported as standard summary-level
evidence tables.

The bundled end-to-end example is a small synthetic demonstration and CI test.
For real studies, use the workflow modules independently and treat each output
table as a reviewed evidence layer.

## Quick Demonstration

```bash
git clone https://github.com/kaiyao28/DrugPipe.git
cd DrugPipe

python3 -m venv .venv
source .venv/bin/activate

make install
make test
make demo
```

The demonstration writes generated outputs to:

```text
results/example/
reports/example/
```

The Python package and CLI entry point are installed as:

```bash
osteo-target-gwas --help
```

## Example Disease

The bundled example uses synthetic osteoporosis, bone mineral density and
fracture-style summary-level data. It includes illustrative loci near genes
commonly discussed in bone biology, but it is not real individual-level data
and should not be interpreted as a biological result.

## Limitations

DrugPipe ranks target hypotheses for research follow-up. It does not prove that
a gene is causal, safe, druggable, or clinically actionable.

Key caveats:

- nearest-gene mapping alone is weak evidence;
- fine-mapping depends on appropriate LD resources;
- colocalisation depends on QTL quality and tissue relevance;
- MR evidence depends on valid instruments and careful interpretation;
- Phe-MR safety scans are incomplete by design;
- druggability annotations do not guarantee therapeutic feasibility;
- all high-priority hypotheses need expert review and experimental validation.
